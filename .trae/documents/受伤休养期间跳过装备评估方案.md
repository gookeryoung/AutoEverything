# 受伤休养期间跳过装备评估方案

## 背景

用户反馈：殖民者在受伤休养时仍在更换装备，导致容易死亡。

**根因**：`CompGearManager.IsDoingMedicalJob()` (`CompGearManager.cs:220-232`) 仅识别 5 种**医生视角**的医疗 Job（`TendPatient`/`TendEntity`/`Rescue`/`TakeToBedToOperate`/`DoBill`+`Bill_Medical`），未识别**伤员视角**的"受伤休养"状态（Pawn 处于 `JobDefOf.LayDown` 在病床接收治疗/免疫力恢复）。

**死亡链路**：
1. 周期性 `CompTick` (`:160`) 对伤员仍触发 `EvaluateWeapon` / `EvaluateApparel` / `EvaluateInventory`
2. `EvaluateWeapon` (`:450`) `JobMaker.MakeJob(JobDefOf.Equip)` + `TryTakeOrderedJob(JobTag.Misc)` 取消当前 `LayDown` Job
3. `EvaluateApparel` (`:629`) `JobDefOf.Wear` 中断休养；且 `:711`/`:740` 直接 `apparel.Remove` 卸下护盾腰带/护甲，立刻降低防护
4. `EvaluateInventory` (`:850`) `TakeCountToInventory` 取药 Job 再次中断休养
5. 休养被打断 → 免疫力/治疗进度受阻 → 重伤者易死

**预期结果**：受伤休养的伤员跳过所有装备评估（周期 `CompTick`、`ForceEvaluate`、`EvaluateInventory`），直到伤愈离开病床。

## 当前状态分析

### `IsDoingMedicalJob()` 现状（CompGearManager.cs:220-232）

```csharp
private bool IsDoingMedicalJob()
{
    var job = Pawn.CurJob;
    if (job == null) return false;
    var def = job.def;
    return def == JobDefOf.TendPatient
        || def == JobDefOf.TendEntity
        || def == JobDefOf.Rescue
        || def == JobDefOf.TakeToBedToOperate
        || (def == JobDefOf.DoBill && job.bill is Bill_Medical);
}
```

**3 处调用点**：
- `CompTick` (`:160`) — 周期性评估入口
- `ForceEvaluate` (`:258`) — 手动/全局触发（含 `AutoExecutor.ExecuteGear` 路径）
- `EvaluateInventory` (`:781`) — 药品评估

### RimWorld 可用 API

- `HealthAIUtility.ShouldSeekMedicalRest(Pawn)` — 是否需要医疗休养（受伤/患病，非单纯睡眠）
- `Pawn.InBed()` — 是否卧床（含医疗床与普通床）
- 两者组合准确识别"受伤休养"，不会误判单纯睡眠

## 提议变更

### 变更 1：新增 `IsRecoveringInBed()` 方法

**文件**：`Source/AutoEverything/AutoEquipment/CompGearManager.cs`

**位置**：`IsDoingMedicalJob()` 方法之后（`:232` 之后）

**新增方法**：
```csharp
// ===================== 伤员休养守卫 =====================

private bool IsRecoveringInBed()
{
    // 受伤/患病休养中：卧床且需要医疗休养
    // 避免换装 TryTakeOrderedJob 取消 LayDown Job，打断免疫力/治疗进度导致重伤者死亡
    return Pawn.InBed() && HealthAIUtility.ShouldSeekMedicalRest(Pawn);
}
```

**设计理由**：
- 与 `IsDoingMedicalJob()`（医生视角）分离，保持单一职责
- `ShouldSeekMedicalRest` 过滤单纯睡眠，仅识别真正需要医疗休养的伤员
- `InBed()` 确保伤员确实在床上（而非正在走向床的途中）

## 假设与决策

1. **不合并 `IsDoingMedicalJob` 与 `IsRecoveringInBed`**：两者语义不同（医生 vs 伤员），保持分离便于维护与日志区分。

2. **不修改 `AutoExecutor.ExecuteGear`**：`ExecuteGear` (`:235`) 仅过滤 `pawn.Dead || pawn.Downed`，但休养 Pawn 不一定 `Downed`。通过 `ForceEvaluate` 内部守卫（`:258`）扩展即可覆盖 `ExecuteGear` 路径，无需在 `ExecuteGear` 重复判断。

3. **不修改 `GearContext.GetContext`**：情境检测识别 `Patient` 状态属于另一议题（影响情境加成，不影响打断问题），本计划不处理。

4. **API 选用 `HealthAIUtility.ShouldSeekMedicalRest`**：原生 API，准确识别需要医疗休养的伤员，过滤单纯睡眠。备选 `pawn.CurJobDef == JobDefOf.LayDown` 过于宽泛（含夜间睡眠），未采用。

5. **不更新项目记忆**：本改动属于 Bug 修复，不引入新的设计约束，无需更新 `project_memory.md`。
