# 护甲纯评分驱动方案

## Context

**问题**：殖民者脱下极佳超织物衬衫，换上属性更弱的人皮衬衫。根因有二：

1. **重甲/轻甲类别硬否决误删装备**：`CompGearManager.RemoveWrongArmorType` 和 `EvaluateApparel` 的护甲偏好硬否决基于 `heavyArmorSharpThreshold=0.4f` 判定重甲，可能误将高护甲衬衫当作"重甲"从 Light 偏好角色身上脱下，再因硬否决无法穿回，最终兜底穿上人皮衬衫。
2. **评分 API 误用**：`ApparelArmorScorer`/`ApparelInsulationScorer`/`ApparelContextScorer` 使用 `gear.def.GetStatValueAbstract(stat, stuff)`，该 API 返回 Def 级抽象值，**不应用品质修正**，导致超织物衬衫（极佳）的护甲优势在评分中消失，与低护甲材料近乎平分。

**用户指示**：忽视重甲/轻甲类别判定，一律按防护能力评分选择，关键角色优先。

**目标**：
- 移除 Heavy/Light 硬否决，所有护甲参与评分
- 修复评分 API，正确反映实例属性（含 stuff + 品质 + HP）
- 关键角色优先：复用现有 `CombatTier` 降序排序（`GlobalAllocator`）+ 评级权重加分

## 修改清单

### 1. 移除护甲偏好硬否决（核心）

**`Source/AutoEverything/AutoEquipment/CompGearManager.cs`**：
- `EvaluateApparel`（L562, L576-581）：删除 `ArmorPreference pref = RoleDetector.GetArmorPreference(role);` 及后续 4 行硬否决（`if (pref == Heavy && !apparelIsHeavy) continue;` 等）。所有护甲进入评分。
- `EvaluateApparel`（L546）：删除 `RemoveWrongArmorType(role);` 调用。
- 删除 `RemoveWrongArmorType` 方法（L747-783）和 `HasMatchingArmorOnMap` 方法（L790-817）——不再需要类别纠错。

**`Source/AutoEverything/Allocation/GlobalAllocator.cs`**（L388, L393-408）：
- 删除 `ArmorPreference pref = RoleDetector.GetArmorPreference(role);`
- 删除重甲判定 `bool isHeavy = ...` 及硬否决/匹配奖励（L393-408）
- 保留评级权重 `(float)pawnTier * 0.5f`（L410-414）——关键角色优先的核心

### 2. 修复评分 API（让护甲值正确反映实例属性）

**`Source/AutoEverything/AutoEquipment/Scoring/Apparels/ApparelArmorScorer.cs`**：
```csharp
// 旧：gear.def.GetStatValueAbstract(StatDefOf.ArmorRating_Sharp, stuff)
// 新：gear.GetStatValue(StatDefOf.ArmorRating_Sharp)
```
移除 `ThingDef stuff = gear.Stuff;`，两个护甲值（Sharp/Blunt）改用实例 API。

**`Source/AutoEverything/AutoEquipment/Scoring/Apparels/ApparelInsulationScorer.cs`**：
同样修复 `Insulation_Cold`/`Insulation_Heat`。

**`Source/AutoEverything/AutoEquipment/Scoring/Apparels/ApparelContextScorer.cs`**：
同样修复 L17/23/29/40/47 的 5 处 `GetStatValueAbstract`。

### 3. BeltAllocator 改为基于角色判定（保留护盾腰带约束）

**`Source/AutoEverything/Allocation/BeltAllocator.cs`**：
- `CollectCandidatePawns`（L167）：`GetArmorPreference(role) != Heavy` → `role != Role.Brawler`
- `ScoreBelt`（L192-203）：参数 `ArmorPreference armorPref` 改为内部判断 `Role role`，护盾腰带对 Brawler +100。候选池已全为 Brawler，逻辑等价。
- L116-118 注释和 `armorPref` 变量更新为基于 `Role`。

**保留**：`ApparelShieldBeltScorer`（非 Brawler + 护盾腰带 → Veto -9999f）、`WeaponTraitScorer` 护盾腰带远程武器 Veto、`RemoveWrongShieldBelt`。这些是装备特性约束（护盾阻挡远程射击），非护甲偏好。

### 4. 保留 ArmorPreference 枚举与 GetArmorPreference 方法

`IsBackRow(role)`（`PawnRole.cs` L243-246）依赖 `GetArmorPreference(role) == Flexible`，用于：
- `WorkAllocator` 狩猎优先分配（L561）
- `SidearmAllocator` EMP 副武器分配（L142）

这些用途与护甲选择无关，**不修改**。`ArmorPreference` 枚举、`GetArmorPreference`、`IsBackRow` 全部保留。

### 5. 清理设置字段与 UI

**`Source/AutoEverything/Core/AESettings.cs`**：
- `heavyArmorSharpThreshold`、`heavyArmorPenaltyForLight`、`lightArmorPenaltyForHeavy`、`heavyArmorMatchBonus`：保留字段定义和 `LookCompat` 读取（存档兼容），但代码中不再使用。注释标记为已废弃。

**`Source/AutoEverything/UI/Dialog_GlobalReallocate.cs`**（L136-141）：
- 移除 `heavyArmorSharpThreshold`、`heavyArmorPenaltyForLight`、`lightArmorPenaltyForHeavy` 的 3 个 UI 滑块。这些设置不再生效。

### 6. 更新 README

- 移除"护甲偏好"表格（L57-73），改为说明：纯护甲值评分驱动，关键角色（高 CombatTier）优先获得高护甲装备。
- 更新"防具评分管线"说明：`ApparelArmorScorer`/`ApparelInsulationScorer` 使用实例 API。

- 更新"全局重配"章节：移除护甲偏好硬否决/匹配奖励描述，保留评级权重说明。

## 关键角色优先机制（已有，无需新增）

`GlobalAllocator.ReallocateApparel` 入口按 `CombatTier` 降序排序殖民者（L348-358），高评级先选装备。同分时 `(float)pawnTier * 0.5f`（L414）让高评级胜出。移除护甲偏好硬否决后，高评级殖民者自然获得护甲评分最高的装备。
