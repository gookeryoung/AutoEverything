# 近战角色定位与远程武器清理方案

## Context

**问题**：坚韧格斗双火（射击无火）的殖民者被错误定位为"轻甲工人"（Worker），且主动配备远程武器。

**根因**：
1. **角色定位只看技能等级，不看兴趣**：`RoleDetector.DetectRole`（PawnRole.cs L123-163）的技能判定分支仅用技能等级（≥8 / <5）区分角色。格斗双火但技能等级 < 8 的殖民者会落到 Worker 或中等技能分支，而非 Brawler。
2. **近战小人配备远程的逻辑未清理**：
   - `WeaponTraitScorer` L51-58：仅 `Brawler` **特质**才 Veto 远程武器，基于技能/兴趣判定的 Brawler 角色不 Veto
   - `WeaponSkillScorer` L35-41：双修角色（射击+近战均有火）给远程武器 +50 偏好分，强制双修偏好远程

**用户指示**：坚韧格斗双火（射击无火）应识别为 Brawler；不再考虑近战小人配备远程，清理此部分逻辑。

## 修改清单

### 1. PawnRole.cs：DetectRole 加入兴趣优先判定

**文件**：`Source/AutoEverything/RoleEvaluation/PawnRole.cs` L123-163

在技能判定分支（else 块）开头加入**兴趣组合优先判定**，在技能等级判定之前：

```csharp
int shooting = pawn.skills.GetSkill(SkillDefOf.Shooting)?.Level ?? 0;
int melee = pawn.skills.GetSkill(SkillDefOf.Melee)?.Level ?? 0;
int medicine = pawn.skills.GetSkill(SkillDefOf.Medicine)?.Level ?? 0;

// 兴趣组合优先判定（天赋倾向比当前技能等级更能反映长期定位）
Passion shootingPassion = pawn.skills.GetSkill(SkillDefOf.Shooting)?.passion ?? Passion.None;
Passion meleePassion = pawn.skills.GetSkill(SkillDefOf.Melee)?.passion ?? Passion.None;

// 近战有火且射击无火 → Brawler（无论技能等级）
if (meleePassion != Passion.None && shootingPassion == Passion.None)
{
    result = Role.Brawler;
    reason = $"近战{GetPassionName(meleePassion)}+射击无火 (兴趣优先)";
}
// 射击有火且近战无火 → Shooter
else if (shootingPassion != Passion.None && meleePassion == Passion.None)
{
    result = Role.Shooter;
    reason = $"射击{GetPassionName(shootingPassion)}+近战无火 (兴趣优先)";
}
// 医生：医疗是其最佳战斗相关技能且 >= 8
else if (medicine >= 8 && medicine >= shooting && medicine >= melee) { ... }
// ... 其余技能等级判定保持不变
```

新增 `GetPassionName` 辅助方法（或复用 WeaponSkillScorer 的同名方法，移至 RoleDetector 公开）。

**效果**：坚韧格斗双火（射击无火）→ Brawler，无论技能等级是否 ≥ 8。

### 2. WeaponTraitScorer.cs：Brawler 角色 Veto 远程武器

**文件**：`Source/AutoEverything/AutoEquipment/Scoring/Weapon/WeaponTraitScorer.cs` L48-64

将"仅 Brawler 特质 Veto 远程"改为"Brawler 角色 Veto 远程"：

```csharp
// Brawler 角色（基于特质或兴趣/技能判定）：拒绝远程武器
// 设计意图：近战定位的小人不应配备远程武器，无论是否有 Brawler 特质
if (role == Role.Brawler && isRanged)
{
    breakdown.Veto(-9000f);
    breakdown.AddScore(Name, "格斗者角色+远程=拒绝", -9000f);
    return;
}

// Brawler 特质：近战武器额外加分（特质偏好，保留）
if (pawn.story.traits.HasTrait(TraitDefOf.Brawler) && isMelee)
{
    breakdown.AddScore(Name, "格斗者特质+近战", 50f);
}
```

移除原 L49-50 注释"仅真正 Brawler 特质才拒绝远程，技能型 Brawler 不拒绝"。

### 3. WeaponSkillScorer.cs：移除双修远程偏好

**文件**：`Source/AutoEverything/AutoEquipment/Scoring/Weapon/WeaponSkillScorer.cs` L35-41, L83-94

删除 `IsDualPassion` 方法和远程武器双修偏好加分：

```csharp
if (isRanged)
{
    SkillRecord shooting = pawn.skills.GetSkill(SkillDefOf.Shooting);
    if (shooting != null)
    {
        float passionMult = GetPassionMultiplier(shooting, weights);
        float score = shooting.Level * weights.w_skill * passionMult;
        string desc = breakdown.CollectItems ? $"射击{shooting.Level} × {weights.w_skill:F1} × {passionMult:F1}({GetPassionName(shooting.passion)})" : null;
        breakdown.AddScore(Name, desc, score);
    }
    // 移除：双修远程偏好 +50 逻辑
}
```

删除 `IsDualPassion` 方法（L83-94）。

### 4. CheckMeleeSidearm 保留（不修改）

`CheckMeleeSidearm`（CompGearManager.cs L877-956）是应对**玩家手动给副武器**的贴身切换逻辑，不是自动分配。Brawler 角色不再自动配备远程武器，但 Shooter 角色仍可能被玩家手动给近战副武器，`CheckMeleeSidearm` 对其仍有意义。**不修改**。

### 5. 更新 README

- **主武器选择规则表格**（L49）：移除"双修（射击+近战均有火）| 远程"行
- **关键约束 1**（L39）：改为"Brawler 角色（基于特质或兴趣/技能判定）拒绝远程武器"

### 6. GetPassionName 辅助方法

`WeaponSkillScorer.GetPassionName`（L73-81）是 private，`DetectRole` 也需要用。两个选择：
- **方案 A**：在 `RoleDetector` 新增 public `GetPassionName`，`WeaponSkillScorer` 改为调用 `RoleDetector.GetPassionName`
- **方案 B**：在 `DetectRole` 内联字符串（$"近战{（meleePassion==Passion.Major?"双火":"单火")}"）

选**方案 B**（内联），避免跨类依赖，KISS 原则。

## 验证

1. `make check` 编译通过，零警告零错误
2. 游戏内测试：坚韧格斗双火（射击无火）殖民者应被识别为 Brawler（日志显示角色变化）
3. Brawler 角色不再配备远程武器（评分 Veto -9000f）
4. 双修角色（射击+近战均有火）按技能等级判定，不再强制偏好远程
